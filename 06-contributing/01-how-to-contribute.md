# How to Contribute

## Start with a Discussion

Have an idea for the spec? Start a
[Discussion](https://github.com/skillbooks-ai/skillbook/discussions). Discussions are the
right place for:

- New ideas and proposals ("What if skillbooks supported X?")
- Questions about intent ("Why does the spec require Y?")
- Use cases that feel unsupported
- Feedback on the format from real-world usage

Discussions are low-friction — no template, no formal process. Just describe your idea
and the community will weigh in.

## File an Issue

Once an idea has consensus (or if you've found a concrete bug), open an
[Issue](https://github.com/skillbooks-ai/skillbook/issues):

- **Spec bugs** — ambiguities, contradictions, or errors in the specification
- **Accepted proposals** — ideas from Discussions that are ready for implementation
- **Missing content** — gaps in the spec that need to be addressed

Include the section affected, what's wrong or missing, and a suggested fix if you have one.

## Submit a Pull Request

For concrete spec changes, submit a PR:

1. MUST reference the related Issue or Discussion
2. Include a diff of the specification changes
3. Explain the rationale — why this change improves the format
4. Follow the existing style: clear, direct language

All PRs require maintainer review before merging.

## Style Guide

Specification language should follow [RFC 2119](https://datatracker.ietf.org/doc/html/rfc2119):

- **MUST** / **MUST NOT** — absolute requirements
- **SHOULD** / **SHOULD NOT** — recommended, with valid exceptions
- **MAY** — optional

Write in clear, direct language. Avoid marketing terms and vague descriptions.

## Code of Conduct

This project follows the [Contributor Covenant](https://www.contributor-covenant.org/version/2/0/code_of_conduct.html).

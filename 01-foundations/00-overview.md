# Foundations

Skillbooks are an extension of the Agent Skills open standard. This section explains what
that means, how skillbooks are structured, and the naming rules that keep everything consistent.

## When to Read This Section

- You're new to skillbooks and want to understand what they are
- You need to understand how skillbooks relate to Agent Skills
- You're setting up a new skillbook project and need the directory layout
- You want to know the naming rules before creating folders and files

## Pages in This Section

- `01-agent-skills.md` — How skillbooks extend the Agent Skills open standard
- `02-directory-structure.md` — Required and optional files, where everything goes
- `03-naming-conventions.md` — Folder, file, and namespace naming rules
- `04-types.md` — The two skillbook types: `reference` and `guide`

Start with `01-agent-skills.md` if you're new. If you're already familiar with Agent Skills,
jump to `02-directory-structure.md` to see how a skillbook is laid out.

# Content

Skillbook content follows an atomic model: each page covers one concept, each section groups
related concepts, and agents fetch only the pages they need.

## When to Read This Section

- You're writing content pages for a skillbook
- You need to understand the `00-overview.md` convention
- You want to know the rules for page length, structure, and cross-references
- You're adding tags to pages for lookup

## Pages in This Section

- `01-section-overviews.md` — The `00-overview.md` convention for section entry points
- `02-content-pages.md` — Page rules, what makes a good page, and what to avoid
- `03-tags-and-index.md` — Optional tags in frontmatter and TAG-INDEX.json for O(1) lookup

Read in order. Section overviews establish the pattern, content pages define the rules,
and tags add optional discoverability.
